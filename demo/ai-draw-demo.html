🎨 AI Draw (WebGPU)

In-browser AI collaborative pixel canvas using WebLLM.
Draw something, then ask the AI what it sees or to improve it.
Fully private, FOSS guardrails enforced. No server required.

How it Works

Canvas as Text

The AI "sees" the canvas because every pixel is serialized into the prompt as a compact text grid. This is efficient for small models because there is no image encoder—just plain text:

................ ................ .....####....... ....#....#...... ....#....#...... ....#....#...... .....####....... ................ ................

Each row is a line of text. . = empty (white), # = filled (black). A 64×64 canvas is about 4 KB of text—well inside an 8K context window.

How the AI Draws

The AI can modify the canvas in two ways:

  • Individual commands: DRAW x y sets a pixel black; ERASE x y sets it white.
  • Full canvas block: The AI outputs [CANVAS] followed by the complete grid and [/CANVAS].

Both methods are parsed automatically and applied to the live canvas in real time.

Context Budget

We use a 1-bit (black & white) palette so the canvas can be larger while staying within the model's context limit. The default is 64×64, which uses roughly 4K tokens for the canvas plus room for conversation. You can switch to 32×32 for even smaller models, or 128×128 if you have headroom.

Browser Caching & Loading Behavior

This component requires the user to explicitly click "Load AI Model" to initiate the WebGPU engine. This prevents hijacking your GPU and network bandwidth immediately upon page load.

What happens when the page is refreshed?

  • The Download is Cached: WebLLM automatically utilizes the browser's native Cache API. After the initial ~1.5GB network download, the model weights are stored securely on your hard drive.
  • VRAM Initialization: A page refresh destroys the active WebAssembly memory and WebGPU context. When you click "Load AI Model" after a refresh, the component skips the network download and rapidly reads the weights from the local cache directly into the GPU's VRAM. This takes only a few seconds.

Acknowledgments, Technologies & Attribution

Building a fully private, in-browser AI collaborator with robust guardrails relies on an incredible ecosystem of open-source tools:

Core AI & Inference
  • WebLLM (@mlc-ai/web-llm): The core inference engine bringing LLM chat to browsers using WebGPU acceleration and WebAssembly.
  • Gemma 2B (Google DeepMind): The quantized gemma-2b-it-q4f16_1-MLC model used here is part of a family of lightweight, state-of-the-art open models.
  • WebGPU API: The modern web standard allowing applications to access the device's underlying GPU.
Security & Guardrails

The deterministic prompt injection scanner relies on regex patterns compiled, tested, and refined by the cybersecurity community. We credit the authors of the following FOSS projects:

  • LlmGuard (North-Shore-AI): Comprehensive security protection for LLM applications.
  • ai-guardian (itdove): A robust security layer for detecting manipulation attempts.
  • llm-prompt-guard (npm package): A sub-millisecond prompt injection firewall designed for TypeScript/JS.
